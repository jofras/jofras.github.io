3D Graphics Pipeline

Built a full interactive 3D graphics pipeline as part of the graphics fundamentals course at Purdue.

Motivation

I’ve always been quite interested in visual things, whether at art class, the movies, or in games. Light shows at concerts, et cetera. I also like linear algebra. So when the opportunity presented itself to do a hands-on graphics course while on exchange (the one at ETH is pretty theoretical), I jumped on it.

Our goal was to start from absolute scratch (blank code files) and build an interactive (i.e., controllable camera) 3D graphics engine by the end of term. It’s one of the harder senior CS courses at Purdue. I hadn’t done too much C++ and my linear algebra was a little rusty, so I got a little nervous.

Design

The course was pretty much purely implementation-based. In lectures, we’d discuss different approaches to each new feature, the prof’s IDE projected up front.

We started off with basic vector and matrix operations; defining Vector and Matrix objects in C++ and basic operations, plus rotations, projections et cetera. In parallel, we set up the framebuffer to view the objects we were going to project. One of the first assignments was creating a few shapes and making part of your name fly across the screen:

Pretty soon after that, we started loading predefined point clouds into our programs and trying to create the wireframes for them. We also added basic camera operations like zooming, panning, and tilts. Here’s a screenshot from the assignment video that week:

Next, we worked on filling in the teapot, as well as more camera operations: moving the camera’s position in space, forward, backward, spinning and sideways. Plus defining interpolated routes combining multiple camera ops.

Now that we had a fully-functional camera and the ability to rasterize objects, it was time to add a (moving) light source. We studied multiple lighting methods based on different normal vectors and so on. It took me a few tries to get it right:

Results

This project taught me so much. Not only about graphics, but about linear algebra, coding in C++, and working with other students, bouncing ideas off each other, comparing engines.

It also taught me to schedule my work better. As I was on exchange, I was travelling a lot, especially given I had Mondays and Fridays off (long weekends every weekend). Deadlines were Wednesdays at noon. A lot of these features were built at 2AM on a Tuesday upon getting back from O’Hare after a weekend trip. But as the semester progressed, I started working on the new projects earlier and earlier in the week, eventually not getting too stressed anymore.

I really enjoyed the course and would recommend it to anyone going to Purdue. It’s some of the most fulfilling work I did during my bachelor’s. It’s cool to have visual feedback instead of (or next to) error messages when debugging.
